A few notes on proof of work

Proof of work historically started with CPU implementations, then with GPU and then with increasingly more efficient ASICS thanks to shrinking and optimizations, unless the algorithm is designed to be somewhat resistant which means it can try to saturate something which then functions as the bottleneck to prevent from benefitting from further optimizations.

  • Memory hard or bound: hardness is easier but technology progressession makes it a long term poor choice, where as being bound by latency means having a physical limits to possible improvements, but it is harder to design and is coupled with the complexity of the algorithm.
