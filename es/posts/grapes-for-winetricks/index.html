uvas para vino

Una configuración basada en capas para recetas de winetricks

Vino se utiliza para ejecutar programas de Windows en Linux. El tiempo de ejecución funciona reimplementando dlls para ser compatible con Linux. Sin embargo, el software no solo necesita las bibliotecas base, existen dependencias que deben instalarse para cada pieza de software.

Wine admite diferentes versiones de Windows ... (¿todas?), Por lo que es necesario utilizar diferentes entornos para garantizar la compatibilidad entre el software y las versiones de Windows.

Algunas dependencias también pueden ser incompatibles entre sí, por lo tanto, el uso de prefijos de vino . El uso de prefijos de vino significa que, por lo general, cada aplicación tiene su propio entorno (como contenedores), lo que significa que todas las dependencias tiene que ser reinstalado para cada nuevo programa. No es así, diferentes programas pueden compartir el mismo entorno, pero como se indicó, pueden surgir problemas difíciles de depurar.

Winetricks automatiza la instalación de dependencias (como los paquetes .NET y DirectX), sin embargo, incluso si están programadas, las configuraciones pueden llevar bastante tiempo.

Y aunque su caso de uso principal es habilitar actualizaciones atómicas para sistemas operativos instalados en bare-metal, podría usarse para compilar prefijos de vino con colecciones preinstaladas de dependencias, como flatpak usa el mismo método para compartir tiempos de ejecución entre diferentes aplicaciones, efectivamente podría haber diferentes flatpak tiempos de ejecución del vino para familias de software que comparten la mayor parte de las dependencias; la única pregunta que queda es si hay algún problema de licencia al distribuir este tipo de tiempos de ejecución con lo que efectivamente sería software de Windows preinstalado . :S
