Almacene su actividad web

No dejes tus pensamientos en plataformas que no te pertenecen

Cuando publica una respuesta en un sitio web o una red social, se deja allí, tal vez durante mucho tiempo, o eventualmente se eliminará. Tener un middleware entre la salida y la entrada que almacena todo lo que escribe localmente es algo que puede parecer inútil a corto plazo, pero a medida que acumula más datos, puede usarse como una base de datos para una búsqueda de texto completo de sus pensamientos, o entrenar a un bot de chat para dar respuestas similar al tuyo.

El problema con dicho software es la interfaz, idealmente desea almacenar el texto después de que esté formateado correctamente, por lo que debería ser un entero Editor WYSIWYG con el que los usuarios interactúan en lugar del editor ofrecido en el sitio web / plataforma. Esto generalmente se siente torpe y opaco, y considerando que el interés a corto plazo de registrar toda su interacción es muy pequeño, una implementación que sea más amigable para la adopción del usuario es difícil.

Existen soluciones para editores como emacs / vim que te permiten escribir dentro del editor y luego enviar el texto al destino, aunque en este caso el interés está en mejorar la experiencia de edición, no en hacer un seguimiento de la salida.
