Mesin telusur

Kaca AMP, kaca turbo, indeksasi ... sing apik, sing ala ...

Kudu cepet!

Kanggo sawetara wektu google wis nyurung kaca AMP. Amarga data kasebut kaping mbukak kaca mengaruhi pengalaman pangguna ... temenan. Iku wis liwat sawetara tambalan kasar nanging ing proyek open source terus mlaku. Nanging nalika nulis isih eksperimen.

Aja Ulangi dhewe

AMP Sejatine nemtokake maneh akeh tag HTML, lan mbatesi atribut apa sing bisa digunakake. Saiki bayangake, sampeyan duwe kaca web, lan sampeyan pengin nawakake versi AMP. Iki dudu sing bisa ditindakake kanthi terprogram , supaya sampeyan kudu ngliwati kabeh kaca, ngerteni apa sing didhukung spek AMP lan nyetel kaca kasebut kanthi bener ... kerumitan banget, iki yen sampeyan pengin menehi pengalaman sing konsisten antarane kaca standar lan kaca AMP.

Aku mutusakÃ© ora kanggo. Dadi aku mung nganggep kaca AMP minangka teks plus CSS. Kabeh liyane aku nyopot. Iki mesthine dudu apa sing dikarepake para AMP devs kanthi tujuan ningkatake pengalaman pangguna...Nanging paling ora aku bisa ngasilake kanthi otomatis.

Saenipun sampeyan pengin nerbitake situs web sampeyan kanthi AMP ing atine wiwit wiwitan, supaya sampeyan tetep konsisten. Nanging yen sampeyan nulis kanggo AMP, sampeyan mung nulis kanggo AMP, sing ora becik kaya taman tembok. Duwe menehi hasil karo HTML lan ampHTML iku akeh churn kanggo njaga. Sawetara AMP komponen web wis digawe bali menyang W3 standar. Nanging akeh sing isih eksperimental lan sapa ngerti yen AMP bakal dadi standar utawa mung Google. Ing cathetan apik baidu katon ndhukung (bebarengan karo sawijining MIPkaca...sanajan ora katon apik...).

Kabeh file iki cache ing server mesin telusuran. Iki uga ora katon apik kanggo aku. Proyek open source AMP paling sethithik kudu menehi a dhewe host-bisaserver kanggo kaca web, sanajan mung kanggo tujuan pangembangan.

Yandex

Yandex uga duwe kaca spin ... turbo dhewe, sing luwih kenceng, amarga sampeyan ora bisa nyakup CSS khusus ing kaca kasebut, nanging sampeyan kudu nyetel saka dashboard webmaster Yandex. Uga kaca turbo dudu kaca. Sampeyan nyedhiyakake feed RSS, utawa file YAML sing gedhe karo kabeh kaca sing pengin ditawakake turbo wujud. Lan Yandex slurps lan serves ing kaca telusuran ... dadi luwih kanggo web crawlable ...

Apa tujuane?

Kaca cepet sing katon apik? ADS sing mbukak cepet? Nelusuri ing sumber? Pola kanggo kula katon kaya integrasi vertikal kabeh komponen sing pindhah saka pangguna, kaca telusuran, lan isi, langsung menyang mesin telusur. Arah kanggo mbangun luwih akeh ing sekitar kebon tembok sing dioptimalake ( gemeter).

Muga-muga aku salah. Aku kaya kanggo ndeleng amp dissolve menyang apa bakal HTML6 utawa kaya, aku mesthi ora nganggo komponen eksperimen. Umume situs web ditelusuri saka seluler. Mbedakake antarane "desktop" lan "seluler" ing babagan kacepetan bisa ngrugekake yen saiki wong nganyarke telpon luwih cepet tinimbang desktop lan laptop, saengga telpon bisa dadi piranti paling cepet. Kaca kudu dirancang kanggo migunani kabeh wektu, pendekatan "responsif", sing bisa mbebani pangembangan situs web cukup akeh, isih luwih apik tinimbang kaca web multi-versi kaca sing kacau.
