Cara kanggo ngakses desktop remot saka tablet Android.

Kuwalitas streaming, konfigurasi input lan tombol cendhek timpa...

Gol

duwe lingkungan iso digunakke ing tablet kanggo program (karo emacs).

Solusi sing jelas (?).

Gunakake tablet minangka klien tipis kanggo persiyapan desktop sing tepat.

protokol kasedhiya

  • VNC lan RDP koyone cukup alon, nanging kudu paling baterei loropaken, RDP kudu strictly luwih apik ing Windows ing paling.

  • Splashtop misale jek nang endi wae ing tengah, ora latensi gedhe lan framerate kurang, nanging kontrol input apik.

  • NOMesin (Protokol NX) katon luwih apik tinimbang splashtop, luwih bisa dikonfigurasi lan fleksibel tinimbang splashtop.

  • cahya rembulan (klien) lan sunshine (server) (Aku duwe AMD R9 290 lawas) katon ok-ish ing 30FPS, dene ing 60FPS wis latency owahan.

  • Parsec . Iki misale jek duwe latensi paling apik (ing 60FPS) nanging aku durung nyoba panggunaan baterei, saiki iki sing digunakake.

Kontrol input

Aku masang keyboard mechanical kanggo tablet, karo Konverter OTG . Bluetooth uga bakal apik ... Masalahe yaiku sawetara peta Android cukup umum trabasan sing digunakake ing windows, sing nggunakake salah sijiMETA (ALT ) utawaMETA (tombol windows). Lan ora menehi cara kanggo mateni. KeyMapper sebagian dianggo, ing sing bisa mateni tumindak saka sisih Android, nanging aku isih ora bisa nerusake keycombinations.

Remote utawa asli?

Apa nggunakake klien remot kabeh apik? Mungkin nggunakake emacs langsung ing Android lan nggunakake kemampuan remot emacs minangka persiyapan sing luwih kompatibel? Kanggo nginstal emacs ing Android sampeyan kudu nggunakake chroots. UserLand nggampangake nggawe chroots saka distribusi umum lan langsung miwiti XServer. Nanging duwe pengalaman emacs sing tepat isih mbutuhake akeh konfigurasi. Apa emacs-nox ing termux luwih apik tinimbang emacs-gtk liwat userland/VNC? Termux duwe dhukungan kanggo aplikasi x11 . Iku nggunakake Xwayland kanggo nyedhiyani support X11 kanggo app, katon luwih cepet saka server xsdl x11, amarga (Aku guess) nggunakake memori sambungan kanggo stream buffering (tinimbang TCP utawa soket unix). Isih ora ana dhukungan akselerasi HW, tabletku uga nggawa GPU POWERVR sing isih durung duwe driver open source (nanging mesthine jadwal bakal dirilis ing endi wae ing Q2 2022). Aku uga wis kasil mbukak a linuxdeploychroot sing dikonfigurasi sing ora duwe overhead proot ing ndhuwur server termux-x11. Nanging chroot kudu cocog karo versi wayland (lan uga mesa) saka termux.

Kesimpulan

Ngelingi biaya konfigurasi kanggo mbukak emacs ing tablet Android, ing wektu iki sesi remot isih luwih trep. Yen upaya X11 termux plus release pungkasan saka driver powervr dadi sukses, imbangan bisa pindhah menyang setelan native.

Yen emacs duwe dhukungan kanggo tumpukan grafis Android, ora bakal dadi pilihan maneh, nanging ora kaya ngono, lan ora ana minat, (ana uga ora ana kapentingan ing antarmuka tutul, mula sing paling apik. pilihan kanggo nggunakake emacs ing piranti genggem tanpa keyboard tepat ing titik iki bakal emacspeak).

NixOnDroid bisa nyuda rasa lara konfigurasi kanggo nambah kabeh perkakas sing dibutuhake dening emacs, utawa dhukungan nix sing bakal teka kanggoarm64.

Isih ana masalah sawetara trabasan keyboard sing ora dilewati, sing solusi umum yaiku map menyang trabasan liyane, untung ana sing kuat banget. aplikasi keymapping android . Aku kudu mriksa ing consuption baterei parsec, lan yen dhuwur banget aku bakal nimbang salah siji RDP kosong, utawa prasaja sesi ssh remot karo emacsi ing terminal.
