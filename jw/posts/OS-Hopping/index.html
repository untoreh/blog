OS Hopping

A ringkesan Brief sistem penggaweyan sing aku wis digunakake

Nalika entuk komputer pribadi sing pertama teka nganggo Windows XP (umur 13 taun). Sawise sawetara kekacoan kudu sinau cara instal maneh windows ... Ora kaya plug and play kaya saiki ... Pengalaman pisanan karo linux yaiku nganggo PCLinuxOS sing umume isih arep amarga nawakake instalasi grafis lengkap. Sawise iku aku langsung mlebu Arch Linux . Aku main karo Arch Linux sawetara wektu (kaya nggawe UI gnome katon kaya MacOS utawa Windows Vista gnome katon mods ), banjur bali menyang Windows amarga game .. Saka Windows, aku ngatur sawetara server linux pribadi nalika mbukak centos / debian. Sawise pirang-pirang taun bali menyang Linux, dhisik aku nyoba Alpine , nanging repositori durung (durung?) kepenak kanggo pengalaman desktop (pulseaudio isih ana ing tes repo).

Banjur aku nyoba Linux Batal kanggo sawetara wektu, nanging aku nggunakake musl versi adhedhasar sing ora nduwe paket dhukungan sing apik, mula aku mbukak linux plus aplikasi ing wadhah ing i3wm lingkungan adhedhasar. Nanging aplikasi grafis kontainer durung nyebar, solusi kaya flatpak snapd utawa firejail durung nyebar utawa malah stabil, mula aku pindhah menyang Ubuntu, lan ngonfigurasi ZFS RAID-Z ( urip mbebayani ) instalasi root. Minangka lingkungan desktop aku isih tetep i3wm, banjur menyang swaywm sabab wayland, banjur plasma KDE amarga aku ora bisa repot maneh karo file file systemd sing kudune pangguna ngatur mung supaya entuk persiyapan sing bisa digunakake kanthi minimal kanthi ubin WMs.

KDE apik banget, lan aku nganggep bisa ngalami getihen ing lingkungan desktop, nanging uga gedhe, lan kanthi akeh aplikasi ora duwe pangopÃ¨nan sing apik, mula saranku supaya ora nyoba banget supaya bisa tetep tetep ing lingkungan GUI K * lan mung gunakake apa sing bisa digunakake, KDE duwe dhukungan sing apik kanggo gaya GTK. KWin duwe opsi pangaturan dhewe sing apik banget lan sampeyan bisa ngubungake tombol hotkey supaya prilaku kasebut cedhak banget karo a ngambang-dhisik ubin WM. Yen sampeyan pengin a kothak-pisanan ubin WM banjur ana sawetara ekstensi, sing ora bisa digunakake kanthi apik ing wayland (nalika nulis).

Akhir-akhir iki aku bali menyang Windows (10) .. maneh , ora utamane amarga game amarga aku isih durung kudu golek wektu lan motivasi kanggo main maneh, nanging rada kepengin weruh a WSL lingkungan linux adhedhasar ditambah karo integrasi sing apik kanggo aplikasi grafis ( WSLg). Nalika nulis, sedhih pengalaman Windows 10 minangka penurunan de-facto saka linux KDE. Windows karo sawijining sentris mouse UX kayane ora ana orientasi tumrap kustomisasi trabasan keyboard sing jero. Ora jelas yen dening Budaya windows trabasan dikuculi, kaya sing katon minangka bolongan keamanan utawa mung watesan saka cangkang windows sing kuwat, sing meksa kustomisasi liwat override registri. Powertoys isih isih butuh cara sing dawa kanggo nyukupi kabeh kabutuhane wong sing kepengin supaya lingkungane cocog karo kebiasaan (ora yakin aku bisa dianggep pangguna kekuwatan ..). Nanging aku bisa dadi fleksibel lan sawetara program pihak katelu lingkungane wis bisa ditampa.
