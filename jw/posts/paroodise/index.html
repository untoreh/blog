Paroodise

Utilitas cilik kanggo modifikasi sistem file langsung

Nalika pisanan ngrampungake panggunaan distribusi linux kanggo server jarak jauh, aku butuh cara kanggo nginstal sistem file root kanthi cepet ing host target, mula aku nulis [paroodise].

Supaya bisa murup piranti pamblokiran boot sampeyan kudu ngilangi . Sampeyan mung bisa ngilangi yen sampeyan ora nggunakake maneh. Kanggo mungkasi nggunakake, sampeyan kudu miwiti maneh layanan ing panggonane, saka sistem file root liyane. Iki padha karo initramfsapa nalika nggawe OS adhedhasar linux, kernel nglakokake gambar boot sing nyetel sistem file saka endi sing sejatine init layanan diluncurake.

Kanggo nggayuh iki ing wis mlaku sistem kita kudu ati-ati kepiye carane miwiti maneh proses. Kita ora bisa mateni ssh kajaba kita yakin skrip bakal sukses nganti sukses.

Proses kabeh luwih gampang ing distribusi sing ora adhedhasar sistem, amarga sistemd pancen ana ing kernel linux, mangling kanthi proses kanthi sembrono bisa nyebabake panik kernel ... nyatane distribusi sing luwih anyar yaiku sing biasane kedadeyan :)

Nalika aku nulis mini utility iki kayane aku ora ngerti cara stabil kanggo ngasilake proses sing ngluwihi sesi ssh asli, Sampeyan kudu ditulis maneh kanthi metode sing luwih konsisten.

Kabeh proses kalebu

  • nyalin file minimal sing dibutuhake kanggo boottrap maneh sistem file ing memori

  • mindhah (utawa nggawe) titik-titik sing dipasang inti (/dev, /proc, /sys)

  • mateni layanan sadurunge sing digawe dening wit proses systemd sing ana

  • nglakokake maneh sistemd menyang sistem file pivoted anyar,systemctl daemon-reexec

Yen kabeh bisa sukses, bisa uga saiki ngasilake layanan ssh anyar lan mlebu ing sesi sing dipasangake titik asli kanggo modifikasi.
