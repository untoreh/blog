Gawe Windows Bearable

Dhaptar alat supaya Windows bisa ngrasakake kaya Linux

Wiwit aku ngalih menyang windows 10 nganggo WSL Aku mikir aku bakal nggawe dhaptar piranti lunak sing kudu digunakake kanggo nggawa windows nyedhaki alur kerja linux sing biasane. Ayo dadi jelas yen Windows UI nyedhot, wis macet ing90s lan kabeh mesthine kudu diklik lan diseret nganggo mouse mouse. Aku ora ngerti yen a budaya bab (windows devs kudu seneng mouse banget), utawa kanggo target demografi ("klik ing bab kasebut!" gampang diterangake), utawa amarga praktik paling keamanan (yen kita kakehan kontrol kanggo pangguna, malware bakal ngeksploitasi) utawa mung Windows Shell digawe saka tumpukan utang teknologi sing luwih angel nambah fungsi kanggo pangguna majeng. Kasunyatane yaiku prekara-prekara sing sederhana ing Linux mbutuhake solusi sing ana ing Windows.

Desktop Virtual

Ing linux saben windows manager duwe sawetara desktop virtual, windows wis dibangun desktop virtual kanthi W10. Masalahe yaiku nalika nulis, sampeyan mung bisa muter kiwa lan tengen karo tombol-tombol, lan ora ana tombol-tombol kanggo pindhah menyang desktop X. Ana iki perpustakaan vd nanging aku durung nemokake piranti lunak sing nggunakake, nanging mung AHK[1] skrip kanthi implementasine sing kurang apik kaya "loop nganti kita ana ing VD sing bener".

Kurang trabasan sing tepat kanggo windows desktop internal nggawe aku meksa nggunakake piranti lunak pihak kaping 3, sing akeh, aku milih dexpot amarga nindakake kabeh prekara sing dakkarepake lan liya-liyane (aku bakal senenggo to X lanmove window to X trabasan). Siji-sijine masalah yaiku dexpot ora bisa ngikat tombol sing wis kaiket ... kaya AHK bisa ngalahake tombol sing digunakake dening aplikasi liyane, dene dexpot ora, dakkira amarga gumantung karo windows API sing beda (utawa [dll]). Workaround yaiku ngrancang cara pintas sing dakkarepake (kayataWin+1 ) kanthi trabasan gratis sing bisa disetel ing dexpot (kayataWin+Shift+F1)

Pemetaan keyboard

Powertoys ngidini kanggo map tombol, lan trabasan, supaya aku bisa peta CapsLock-> LeftControl lan RightControl-> LeftControl. Lan trabasan kayaWin+hjkl kanggo panah. Ala bangetWin+l minangka trabasan default windows kanggo ngunci layar lan ora bisa dibaleni maneh ... tombol registri ngatasi trabasan lan ndandani iki ... nganti sampeyan nganyari windows ... dadi sampeyan kudu ngetrapake perbaikan ing saben boot!

Peluncur

Peluncur aplikasi sing disedhiyakake dening powertoys pancen apik banget, ora nggunakake memori luwih akeh tinimbang KRunner lan cepet lan responsif. Siji keluhan yaiku yen kala-kala ngilangi fokus, nanging iki kemungkinane masalah karo Windows sing lara (kaya pervert) kontrol jendhela sing mesthine bakal fokus ing wektu tartamtu.

Tiling

Aku ngalih saka sway menyang KDE ing Linux, mula aku dudu pacangan windows jubin. Powertoys duwe FancyZones sing dandan cilik tinimbang windows dhasar Snaps sing ditawakake jendhela, kaya sing diwenehake kesenjangan lan tata letak. Nanging isih kurang prekara sing paling penting sing ndadekake ubin migunani, yaiku aturan kanggo aplikasi ing windows sing cocog, nanging iki uga butuh dhukungan kanggo desktop virtual ... lan kita wis nyebutake kahanane saiki. Ing Comparison KWin ngidini sampeyan cocog karo windows kanthi definisi sing rumit. FancyZones ngatasi masalah aturan kanthi ngreksa posisi windows ing sesi, nanging aku durung nate mriksa babagan kerjane iki karo desktop virtual sing beda.

Seret windows

Windows mung bisa nyeret saka judhul, lan tambahan sarana kudu digunakake kanggo mindhah nganggo kombinasi trabasan.

Monitor sistem

Kanthi KDE sampeyan duwe widget plasma kanggo nampilake informasi sistem, sanajan widget plasma cenderung nggunakake akeh memori dadi aku ora nggunakake akeh. Kanthi Windows, ora ana fasilitas internal kanggo iki, sawise nyoba sawetara utilitas, aku bisa ngrampungake Monitor Lalu Lintas sing menehi jaringan, CPU, memori, lan bubar ditambahake GPU lan temps. Layanan Sistem

Akeh sing ora seneng karo sistemd nalika ana ing kernel linux..Kanggo Windows, mula aku ilang karo dheweke..Jasa Windows dudu tujuan kanggo pangguna kaya sistemd. Windows duwe Penjadwal Tugas kanggo ngatasi prekara sing katon kayaone-shotfile unit, nanging maneh antarmuka (utawa ora ana) sing nggegirisi. Untunge, kita ora butuh daemon khusus ing windows, amarga umume aplikasi windows sing digunakake mesthine wajar mbukak nalika wiwitan jenis logika, lan ora butuh konfigurasi sing luwih maju. Nyatane, loro sampah alat ( nggrundel ) supaya windows bisa digunakake kaya lingkungan modern, mula mung aplikasi windows asli liyane sing digunakake yaiku browser ( kembang api ) amarga percepatan GPU browser ora apik ing WSL (uga ing linux asli ...) lan pamuter video ( mpv ) .. lan game mesthi ...

Paket

Aku ora seneng coklat amarga mbutuhake hak istimewa administrator, aku luwih dhisik golek paket scoop amarga wis diinstal ing folder pangguna, sing luwih trep lan luwih konsisten, lan nyederhanakake cadangan.

WSL / g

Aku kepencut kanggo ngalih menyang windows bubar WSLg nganyari Kang mbutuhake driver mesa dikompilasi karod3d12 dhukungan backend kanggoopengl . Windows nyedhiyakake lapisan ubuntu pratinjau komunitas, nanging aku milih lengkungan linux amarga wis ana paket AUR kanggo mesa karo d3d12 . Amarga WSL saiki ora ndhukung sistemd, mula aku nggunakake pengawas kanggo ngatur sawetara layanan. Kanggo nggawe manawa supervisor aktif, priksa ing profil cangkang file kunci sing kudu digawetmpfs yen pengawas sadurunge wis diwiwiti.

WSL mung ndhukungext4 sistem file, kanggo nggunakake sistem file liyane, sampeyan kudu masang partisi (utawa disk) kanthi langsung njero WSL VM. Nanging, dheweke kudu dipasang kanthi manual minangka administratorsaka windows. Kanggo otomatis iki, kita bisa nggunakake panjadwal tugas sing ngidini ngliwati UAC kanthi cepet kanthi mbukak tugas kanthi priviledge paling dhuwur . Kita bisa mbukak wsl--mount printah kanggo masang disk / sekat sing dikepengini lan banjur jalanake sistem file saka njero linux. Amarga Pendhaftaran/home/ manggon ing dipasangbtrfs filesystem Aku kudu masang kanthi otomatis, mula digunakake/etc/fstab kanggo peta pemisahan kita (deningLABEL ) kanggo/home . Amarga kita nggunakake nix karonix-env kita kudu ngubungake nyimpen nix menyang/nix ing saben wiwitan wsl, lan amarga/tmp ora aktiftmpfs kita kudu nggawe mount overlay sing bisa dipasangtmpfs liwat ana/tmp direktori, kanggo ngreksa file penting, khusus file X11, sing dibutuhake kanggo ngobrol karo server X.

Gunung kasebut dieksekusi kanthi skrip, sing kudu mbukak sawise kita wis nginstal disk ing njero linux. Kita butuh tugas sing mlaku sawise tugas pasang rampung, lan amarga WSL nggunakake macem-macem VM kanggo pangguna windows sing beda, skrip kasebut kudu mbukak Dudu kanthi hak istimewa sing paling dhuwur (yen ora bakal dipasang ing Administrator VM).

Kanggo mesthekake WSLg bisa digunakake, kita kudu mriksa XDG RUNTIME DIR disetel, amarga beda lan dununge (kanthi standar) ing/mnt/wlsg/runtime-dir . Mangkene tampilan windows ing wayland, sampeyan bisa ngerti manawa kanthi dekorasi jendhela wayland bisa ditarik dening komposer utawa aplikasi, dheweke tetep tetep ngatur konfigurasi GTK / QT, dadi sampeyan bisa ndeleng jendela Linux sing katon asli ing njero cangkang Windows ... sing wiwitane ora ngganggu.

Emulator terminal

Ing linux sing digunakake kucing , amarga iku salah sawijining emulator terminal sing luwih anyar kanthi akselerasi GPU[2]lan mode daemon sing rada stabil sing ngidini pirang-pirang windows kanthi conto sing padha. Nanging ing windows, aku nganggo sawetara terminal adhedhasar windows kaya Windows Terminal, terminal lancar, wsltty, conemu, nanging pungkasane aku ganti wezterm . Sanajan dadi terminal sing luwih anyar, kabeh fitur sing sampeyan pengin saka emulator terminal:

  • tab lan panel, kayata sampeyan ora butuh tmux

  • multiplexing (mode tanpa headless), dadi ora prelu tmux

  • mode salinan teks kanthi keybindings vi, kayata sampeyan ora butuh tmux

  • pilih cepet, mode petunjuk kanggo milih obyek teks kanthi cepet saka scroll-back

  • printah standar kanggo tab anyar

  • Kustomisasi peta kunci sing apik banget, kajaba sampeyan ora bisa nggunakake Kiri / Kanan Nggeser / Ctrl / Alt minangka modifikasi sing beda.

Kanthi Windows, aku uga ngowahi alur kerja supaya ora butuh terminal kanthi mode daemon sing ndhukung macem-macem windows sing beda, kanthi nggunakake terminal dropdown. Kanggo nggawe wezterm nyelehake sawise nyoba akeh script AHK drop-down, akhire aku nemokake siji sing bisa digunakake kanthi cukup. Nanging, alternatif sing luwih apik gempa windows-terminal, nanging aku duwe masalah karo tumindak sing salah karo desktop virtual sing dikelola dening dexpot, dene mitty-quake-console kayane kompatibel karo desktop virtual dexpot. Paling kamungkinan nalika pengalaman desktop Windows internal wis saya apik, aku bakal pindhah menyang windows-terminal-gempa. WezTerm uga lintas platform sing artine, aku bisa terus nggunakake instalasi linux asli. Iki minangka temuan sing apik banget kanggo saklar Windows + WSL iki, amarga Kitty durung ndhukung Windows, lan alternatif terminal liyane, ngalami gangguan sing cukup gedhe Aku ora bisa ngrampungake.
