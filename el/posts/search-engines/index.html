Μηχανές αναζήτησης

Σελίδες AMP, σελίδες turbo, ευρετηρίαση...το καλό, το κακό...

Πρέπει να πάω γρήγορα!

Για λίγο η google πιέζει για σελίδες AMP. Γιατί αυτό λένε τα δεδομένα χρόνους φόρτωσης σελίδας επηρεάζουν την εμπειρία του χρήστη...προφανώς. Έχει περάσει από μερικά τραχιά μπαλώματα αλλά το έργο ανοιχτού κώδικα συνεχίζει. Αλλά τη στιγμή της γραφής είναι ακόμα πειραματικό.

Μην επαναλάβετε τον εαυτό σας

Το AMP ουσιαστικά επαναπροσδιορίζει μια δέσμη ετικετών HTML και περιορίζει τα χαρακτηριστικά που μπορείτε να χρησιμοποιήσετε. Τώρα φανταστείτε ότι έχετε μια ιστοσελίδα και θέλετε να προσφέρετε μια έκδοση AMP. Αυτό δεν είναι κάτι που μπορεί να γίνει προγραμματικά , επομένως θα πρέπει να περάσετε από όλες τις σελίδες, να μάθετε τι υποστηρίζουν οι προδιαγραφές AMP και να προσαρμόσετε τη σελίδα αναλόγως...υπερβολική ταλαιπωρία, αυτό είναι εάν θέλετε να προσφέρετε μια συνεπή εμπειρία μεταξύ της προεπιλεγμένης σελίδας και της σελίδας AMP.

Αποφάσισα να μην το κάνω. Επομένως, αντιμετωπίζω τις σελίδες AMP ως κείμενο συν CSS. Όλα τα άλλα τα αφαιρώ. Σίγουρα δεν ήταν αυτό που είχαν στο μυαλό τους οι προγραμματιστές AMP με στόχο τη βελτίωση της εμπειρίας χρήστη...Αλλά τουλάχιστον μπορώ να τα δημιουργήσω αυτόματα.

Ιδανικά, θα θέλατε να δημοσιεύσετε τον/τους ιστότοπό σας/τους ιστοτόπους σας έχοντας υπόψη το AMP από την αρχή, έτσι ώστε να διατηρείτε τα πράγματα συνεπή. Αλλά αν γράφετε για AMP, είστε μόνο γράφοντας για AMP, κάτι που δεν είναι ωραίο καθώς μοιάζει με περιφραγμένο κήπο. Έχοντας να ασχοληθεί με HTML και ampHTML είναι πολύ δύσκολο να διατηρηθεί. Κάποια AMP εξαρτήματα ιστού έχουν κάνει πίσω στο W3 πρότυπα. Αλλά το μεγαλύτερο μέρος είναι ακόμα πειραματικός και ποιος ξέρει αν το AMP θα γίνει τυπικό ή απλώς ένα πράγμα της Google. Στην καλή νότα baidu φαίνεται να το υποστηρίζει (μαζί με το MIPσελίδες...αν και δεν φαίνονται καλά συντηρημένες...).

Όλα αυτά τα αρχεία είναι αποθηκευμένο στην κρυφή μνήμη στους διακομιστές της μηχανής αναζήτησης. Και αυτό δεν μου φαίνεται εντάξει. Το έργο ανοιχτού κώδικα AMP θα πρέπει τουλάχιστον να προσφέρει α αυτοφιλοξενούμενοςδιακομιστή για τις ιστοσελίδες, ακόμη και για λόγους ανάπτυξης.

Yandex

Yandex έχει επίσης τις δικές του spin...turbo σελίδες, οι οποίες είναι ακόμα πιο αυστηρές, αφού δεν μπορείτε να συμπεριλάβετε προσαρμοσμένο CSS στη σελίδα, αλλά πρέπει να το ρυθμίσετε από τον πίνακα ελέγχου των webmasters του Yandex. Επίσης, οι σελίδες turbo δεν είναι στην πραγματικότητα σελίδες. Παρέχετε μια ροή RSS ή ένα τεράστιο αρχείο YAML με όλες τις σελίδες που θέλετε να προσφέρετε σε αυτές τούρμπο μορφή. Και η Yandex το σβήνει και το σερβίρει στις σελίδες αναζήτησής του...τόσο για έναν ιστό με δυνατότητα ανίχνευσης...

Ποιος είναι ο στόχος;

Γρήγορες σελίδες που φαίνονται καλές; ΔΙΑΦΗΜΙΣΕΙΣ που φορτώνουν γρήγορα; Παρακολούθηση στην πηγή; Το μοτίβο για μένα μοιάζει με κάθετη ενοποίηση όλων των στοιχείων που πηγαίνουν από τον χρήστη, τη σελίδα αναζήτησης και το περιεχόμενο, απευθείας στη μηχανή αναζήτησης. Η κατεύθυνση είναι να χτίσουμε όλο και περισσότερα γύρω από βελτιστοποιημένους κήπους με τοίχους ( θρύψαλα).

Ελπίζω να κάνω λάθος. Θα ήθελα να δω τον ενισχυτή να διαλύεται σε αυτό που θα ήταν HTML6 ή κάτι τέτοιο, σίγουρα δεν υιοθετώ πειραματικά στοιχεία. Το μεγαλύτερο μέρος του ιστότοπου περιηγείται από κινητό. Η διάκριση μεταξύ "επιτραπέζιου υπολογιστή" και "κινητού" ως προς την ταχύτητα μπορεί να είναι επιζήμια εάν σήμερα οι άνθρωποι αναβαθμίζουν τα τηλέφωνά τους πιο γρήγορα από τους επιτραπέζιους και φορητούς υπολογιστές τους, έτσι ώστε το τηλέφωνό τους να καταλήξει να είναι η ταχύτερη συσκευή τους. Οι σελίδες θα πρέπει να σχεδιάζονται για χρηστικότητα όλη την ώρα, η «ανταποκρινόμενη» προσέγγιση, η οποία μπορεί να επιβαρύνει αρκετά την ανάπτυξη ιστοτόπων, εξακολουθεί να είναι μια καλύτερη αντιστάθμιση από τις χαοτικές ιστοσελίδες πολλαπλών εκδόσεων.
