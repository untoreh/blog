OS Hopping

สรุประบบงานคร่าวๆ ที่เคยใช้

เมื่อฉันได้คอมพิวเตอร์ส่วนบุคคลเครื่องแรกมา มันมาพร้อมกับ Windows XP (อายุ 13 ปี) หลังจากมีปัญหานิดหน่อยก็ต้องเรียนรู้วิธีติดตั้ง windows ใหม่...มันไม่ได้เป็นแบบ plug and play อย่างทุกวันนี้... ประสบการณ์ครั้งแรกของฉันกับ linux คือกับ PCLinuxOS ซึ่งเห็นได้ชัดว่า ยังคงเกิดขึ้น เนื่องจากมีการติดตั้งแบบกราฟิกเต็มรูปแบบ หลังจากนั้นฉันก็ตรงเข้าไป Arch Linux . ฉันเล่นกับ Arch Linux มาระยะหนึ่งแล้ว (เช่นทำให้ gnome UI ดูเหมือน MacOS หรือ Windows Vista ผ่าน gnome ดู mods ) จากนั้นกลับไปที่ Windows เพราะเกม.. จาก Windows ฉันจัดการเซิร์ฟเวอร์ linux ส่วนบุคคลบางตัวในขณะที่ใช้ centos/debian หลังจากกลับไปใช้ Linux มาหลายปี ฉันก็ลองใช้เป็นครั้งแรก อัลไพน์ แต่ที่เก็บข้อมูลยังไม่เป็นที่พอใจ (ยัง?) สำหรับประสบการณ์เดสก์ท็อป (pulseaudio ยังอยู่ในการทดสอบ ซื้อคืน)

จากนั้นฉันก็พยายาม โมฆะ Linux ชั่วขณะหนึ่ง แต่ฉันใช้ มุสล เวอร์ชันพื้นฐานซึ่งไม่มีแพ็คเกจสนับสนุนที่ดี ดังนั้นฉันจึงใช้งาน void linux plus apps ในคอนเทนเนอร์บน i3wm สภาพแวดล้อมตาม แต่แอปพลิเคชันกราฟิกแบบคอนเทนเนอร์ยังไม่แพร่หลาย โซลูชันเช่น flatpak snapd หรือ firejail ยังไม่แพร่หลายหรือเสถียร ดังนั้นฉันจึงเปลี่ยนไปใช้ Ubuntu และตั้งค่า ZFS RAID-Z ( อยู่อย่างอันตราย ) การติดตั้งรูท เนื่องจากสภาพแวดล้อมเดสก์ท็อป ฉันยังคงเก็บ i3wm ไว้ จากนั้นจึงไปที่ swaywm ทำให้เกิด wayland แล้วก็ KDE plasma เพราะผมไม่อยากยุ่งกับไฟล์ systemd unit ที่ต้องใช้อีกต่อไป ผู้ใช้ จัดการเพียงเพื่อให้ได้การตั้งค่าการทำงานที่คาดหวังน้อยที่สุดด้วย WM ที่เรียงต่อกัน

KDE นั้นยอดเยี่ยม และฉันคิดว่ามันเป็นขอบที่ขาดไม่ได้ของสภาพแวดล้อมเดสก์ท็อป อย่างไรก็ตาม มันก็ใหญ่เช่นกัน และด้วยแอพจำนวนมากที่ขาดการบำรุงรักษาที่ดี ดังนั้นคำแนะนำของฉันคืออย่าพยายามมากเกินไปที่จะยึดติดกับสภาพแวดล้อม GUI แบบ K* เท่านั้นและ เพียงแค่ใช้สิ่งที่ได้ผล KDE รองรับสไตล์ GTK ได้ดี KWin มีตัวเลือกการปรับแต่งที่ดีมาก และคุณสามารถผูกปุ่มลัดเพื่อให้ทำงานใกล้เคียงกับa ลอยก่อน ปูกระเบื้อง WM หากคุณต้องการ กระเบื้องก่อน การปูกระเบื้อง WM นั้นมีส่วนขยายอยู่สองสามรายการ ซึ่งอย่างไรก็ตาม ใช้งานไม่ได้ดีนักบนทางบก (ในขณะที่เขียน)

เมื่อเร็ว ๆ นี้ฉันได้เปลี่ยนกลับเป็น Windows (10).. อีกครั้ง , ไม่ได้เป็นหลัก เพราะเกมส์ เพราะยังต้องหาเวลาและแรงจูงใจในการเล่นอีกครั้ง แต่ค่อนข้างจะอยากรู้อยากเห็นสำหรับ a WSL สภาพแวดล้อม linux พื้นฐานควบคู่ไปกับการรวมที่ดีสำหรับแอพกราฟิก ( WSLg). ในขณะที่เขียน ประสบการณ์ Windows 10 ที่น่าเศร้าคือการดาวน์เกรดโดยพฤตินัยจาก linux KDE หน้าต่างที่มี เมาส์เป็นศูนย์กลาง ดูเหมือนว่า UX จะไม่ได้เน้นไปที่การปรับแต่งแป้นพิมพ์ลัดอย่างลึกล้ำ ไม่ชัดเจนว่าโดย วัฒนธรรมหน้าต่าง ทางลัดถูกขมวดคิ้วเมื่อถูกมองว่าเป็นช่องโหว่ด้านความปลอดภัยหรือเป็นเพียงข้อ จำกัด ที่รุนแรงของ windows shell ที่บังคับให้ปรับแต่งผ่านการแทนที่รีจิสทรี Powertoys ยังมีหนทางอีกยาวไกลที่จะตอบสนองทุกความต้องการของคนที่ต้องการปรับสภาพแวดล้อมให้เข้ากับนิสัย (ไม่แน่ใจว่าจะถือว่าเป็นผู้ใช้ที่มีอำนาจ..) อย่างไรก็ตาม ฉันสามารถยืดหยุ่นได้มาก และด้วยโปรแกรมของบุคคลที่สามสองสามโปรแกรม สภาพแวดล้อมก็เป็นที่ยอมรับได้ค่อนข้างดี
