เครื่องมือเล็กน้อย

รวบรวมสาธารณูปโภคเล็กๆ น้อยๆ ทำเพื่อขีดข่วน...

action5

[Action5] เป็นเครื่องมือเปล่าเพื่อระบุไฟล์ที่แก้ไขผ่านเช็คซัมในไดเร็กทอรี มีเครื่องมือมากมายที่ทำสิ่งนี้ ดังนั้นฉันไม่แน่ใจ (จำ) ว่าทำไมฉันถึงเขียนมัน

  • เรียกซ้ำในโฟลเดอร์ ดัมพ์ทรีของเช็คซัม (ดัชนี)..

  • ..ทำซ้ำเพื่อดูว่ามีอะไรเปลี่ยนแปลงบ้าง

จุดสำคัญของเครื่องมือนี้คือมันเป็นเพียงคำสั่งเชลล์ ดังนั้นคุณสามารถเสียบคำสั่งรวมการตรวจสอบใดๆ ที่คุณต้องการ โดยค่าเริ่มต้น โปรแกรมจะใช้md5.

bash-utils

วัตถุประสงค์ของ บาชูติลส์ คือการพึ่งพา bash เป็นหลักในการดำเนินการบางอย่างกับกระบวนการ

  • การอ่านไฟล์ภายในตัวแปร (gobbler ) ต้องตรวจสอบให้แน่ใจว่าไฟล์ไม่มีค่า null ดังนั้นจึงต้องจัดเก็บด้วยรูปแบบการเข้ารหัสที่ปราศจากค่า null หมายเหตุ: การใช้เครื่องหมายคำพูดใน bash อย่างไร้กังวล (ในสถานที่ที่ไม่ต้องการ) อาจทำให้หน่วยความจำจำนวนมากสูญเปล่า

  • การตรวจสอบ cpu ในระบบ linux ต้องอ่าน/proc หยุดชั่วคราวโดยไม่ต้องฟอร์ก และคำนวณสถิติด้วยการดำเนินการเชลล์เพียงอย่างเดียว ($(()))

  • การจำกัดกระบวนการต้องส่งอย่างต่อเนื่องSIGSTOP และSIGSTOP สัญญาณนี้ช้ามากในการทุบตี :) เนื่องจากเป็นการวนซ้ำและซอฟต์แวร์อาจทำงานโดยไม่คาดคิดเมื่อหยุดชั่วคราวและกลับมาทำงานต่อเนื่องจากโปรแกรมเมอร์มักไม่คาดหวังว่าซอฟต์แวร์ของตนจะทำงานเป็นระยะ!

dist-haproxy

NS สร้างสคริปต์ สำหรับ haproxy ซึ่งเป็นเครื่องมือที่ดีที่สุดในการส่งต่อการเชื่อมต่อ มีน้ำหนักเบา และมีเวอร์ชันที่ลิงก์แบบสแตติกมีประโยชน์มาก
