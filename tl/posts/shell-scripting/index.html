shell scripting

Mga saloobin sa shell scripting

Bash/ zsh shell scripting nakakakuha ng trabaho tapos na. Medyo pangit din ito. Marahil nilayon ito. Mas kaunti ang nais mong tingnan ito mas maikli ang mga script. Hindi ako tagahanga ng paggamit ng bash / zsh bilang isang interactive shell, ang dami ng mga plugin na kinakailangan upang makabawi para sa isang magandang karanasan ay nakakatakot, at ang bawat karagdagang plugin ay isang pitfall para sa mga posibleng pagbagal ng shell. Huwag kailanman-ang-mas mababa ang halaga ng trabaho tapos na sa pamamagitan ng psprint sa zinit at mga kaibigan ay napaka-kahanga-hanga (at uri din ng tanging matinong pagpipilian para sa isang plugin manager kung nais mong gamitin ang zsh bilang isang interactive shell). Gayunpaman hindi nito nalulutas ang katotohanang lahat ng mga na-load na plugin ay hindi naibahagi memorya sa gitna ng (marahil) maraming mga proseso ng shell.

Shell ng isdaang aking ginustong shell sa oras ng pagsulat. Ang halaga ng mga naka-install na plugin ay napakaliit, dahil ginagawa nito ang karamihan sa mga bagay na nais mong mula sa isang shell sa labas ng kahon . Ang scripting ay medyo mas mabagal kaysa sa zsh ngunit walang nagmamalasakit sa bilis ng pag-scrip kaya maayos ito. Ang syntax ay maganda sa pamana ng ML (tulad ni Julia) na may mga huling pahayag, sa halip na mga kulot na brace. Ang pagsusulat ng mga script ay mas maraming salita kaysa sa bash / zsh subalit ito rin ay mas maraming nababasa. Ang mga bagay tulad ng pagpapalit ng proseso, interpolation ng mga string at piping ay nagtatapos nang kaunti pa. Mayroon ito pandaigdigan variable, ibinahagi sa lahat ng tumatakbo na proseso, at unibersal mga variable, ibinahagi sa mga session, na ginagawang mas madali ang mga bagay nang hindi kinakailangang isulat ang lahat at mapagkukunan ng mga nakakalimutang script.

Elvish ay isang shell na may suporta para sa maraming mga bagay, tulad ng isang daemon o web mode na pinapayagan itong magpadala ng mga utos mula sa malayo ...? Sa palagay ko sobra ito at sinisimulan kong tawagan itong namamaga. Ang IMHO ng isang shell ay dapat magsikap para sa pagiging simple, at magkaroon ng sarili. Sa palagay ko ang mga shell ay dapat na idinisenyo upang mahawakan ang mga pangyayari kung saan ang mga ito ay proseso na nagsimula ng isang kapansin-pansin na bilang ng mga oras, na kumilos nang tuluy-tuloy mula simula hanggang matapos at na may sariling kakayahan na kaya nilang hawakan ang mga pag-crash nang kaaya-aya. Kaya sa kabila ng lahat ng iba pang mga tampok (tulad ng nakabalangkas na data) ... ang mga daemon ay hindi pupunta para sa akin.

Ion Shell ay halos posix shell mula sa redox, na gumagana din sa unix, na may layunin na dinnagtatrabaho sa windows, kahit na hindi ito gumagana ATTOW. Ang layunin nito ay maging isang mas mahusay na shell, kaya't kinukuha nito ang mabuti mula sa posix at idinagdag dito. Mukhang nakatuon ito sa pagbibigay ng isang mahusay na wika ng pag-script, ngunit hindi isang napakahusay na karanasan sa interactive, sinira ako ng shell ng isda, at hindi ako babalik sa isang shell nang wala primera klase pagkakaugnay.

Shell ng langis ay may layunin na palitan ang bash, na kung saan ay kapuri-puri, subalit target lamang nito ang mga unix system na walang Windows na nakikita. Nakatuon ito sa pag-aayos ng bash layer ng pagiging tugma hanggang ngayon, na may mga plano sa hinaharap na mag-focus sa bahagi ng kakayahang umangkop. Isinasaalang-alang ang katotohanan na ang interactive na bahagi ng shell ay hindi pa nakasulat, ginagawang medyo nagdududa ako tungkol sa hinaharap nito at bagaman inaasahan kong hindi ito, pinapakita ko na maaaring maluwag ang singaw bago nito makamit ang mga nilalayon nitong layunin. Gayundin ang kakulangan ng suporta sa windows ay hindi perpekto.

Rash isang shell na may suporta para sa nakabalangkas na data na gumagana sa Raketa mga bagay, kapag tiningnan ko ang raket higit pa susubukan ko ito, kahit na ang aking nakaraang karanasan sa [ shell ng emacs ] ay hindi talaga maganda, dahil ang eshell gumagana lamang sa loob ng emacs, at hindi gumana nang maayos bilang isang shell ng system ... maliban kung ang iyong system ay emacs (sa kabila ng mga emac ay ang aking pangunahing editor at nagsusulat ako ang ilan lisp, hindi ito ang aking operative system). (Narito dapat ko ring banggitin ang Julia REPL, ang pinakamahusay na repl para sa anumang wika ng programa).

Power shell Ang Powershell, ang pangunahing Windows shell ay gumagana rin sa linux at macs, salamat sa cross platform NET . Ngunit ito ay masyadong pandiwang, mabagal, at marahil (hindi na-verify) na magkalat windowsism ; Ito ay isang malaking hindi para sa akin.

Nu shell... syempre tinitira namin ang pinakamahusay para sa huli. Marahil ito ang aking susunod na pumunta sa shell; pansin sa unang klase para sa kakayahang umangkop, maingat na pagsasaalang-alang ng bawat built-in na shell at cross platform, at nakabalangkas na teksto.

Ang mga shell na wala sa listahang ito ATTOW ay marahil ay itinuturing na hindi sapat na kagiliw-giliw.
