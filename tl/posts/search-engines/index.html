Mga search engine

Mga AMP page, turbo page, indexing...ang mabuti, ang masama...

Kelangan kong bilisan!

Matagal nang itinutulak ng google ang mga AMP page. Dahil ang data ay nagsasabi na mga oras ng pag-load ng pahina makakaapekto sa karanasan ng user...malinaw naman. May mga pinagdaanan na magaspang na mga patch ngunit ang open source na proyekto nagpapatuloy. Ngunit sa oras ng pagsulat ay eksperimental pa rin.

Huwag Ulitin ang Iyong Sarili

Karaniwang nire-redefine ng AMP ang isang grupo ng mga HTML tag, at pinaghihigpitan kung anong mga attribute ang maaari mong gamitin. Ngayon isipin, mayroon kang webpage, at gusto mong mag-alok ng bersyon ng AMP. Ito ay hindi isang bagay na maaaring gawin sa pamamagitan ng programming , kaya kailangan mong suriin ang lahat ng page, alamin kung ano ang sinusuportahan ng mga spec ng AMP at ayusin ang page nang naaayon...masyadong abala, ito ay kung gusto mong magbigay ng pare-parehong karanasan sa pagitan ng default na page at ng AMP page.

Nagpasya akong huwag. Kaya tinatrato ko na lang ang mga AMP page bilang text at CSS. Lahat ng iba ay hinuhubaran ko. Tiyak na hindi ito ang nasa isip ng mga AMP devs sa layuning pahusayin ang karanasan ng user...Ngunit kahit papaano ay awtomatiko ko silang mabuo.

Pinakamainam na gusto mong i-publish ang iyong website na may AMP sa isip mula sa simula, upang mapanatili mong pare-pareho ang mga bagay. Ngunit kung nagsusulat ka para sa AMP, ikaw ay lamang pagsusulat para sa AMP, na hindi maganda dahil sa pakiramdam na parang isang pader na hardin. Kailangang harapin ang HTML at ampHTML ay maraming churn upang mapanatili. Ilang AMP mga bahagi ng web nakabalik na sa W3 mga pamantayan. Pero karamihan pa rin eksperimental at sino ang nakakaalam kung ang AMP ay magiging isang pamantayan o isang bagay lamang sa google. Sa magandang tala baidu tila sinusuportahan ito (kasama nito MIPpages...bagama't hindi maganda ang hitsura ng mga ito...).

Ang lahat ng mga file na ito ay naka-cache sa mga server ng search engine. Mukhang hindi rin ito okay sa akin. Ang proyektong open source ng AMP ay dapat mag-alok man lang ng isang self-host-ableserver para sa mga web page, kahit na para lamang sa mga layunin ng pag-unlad.

Yandex

Yandex ay mayroon ding sariling spin...turbo page, na mas mahigpit, dahil hindi mo maaaring isama ang custom na CSS sa page, ngunit kailangan mong itakda ito mula sa dashboard ng Yandex webmasters. Gayundin ang mga turbo na pahina ay hindi talaga mga pahina. Nagbibigay ka ng isang RSS feed, o isang malaking YAML file kasama ang lahat ng mga pahinang gusto mong ialok sa kanila turbo anyo. At hinihigop ito ng Yandex at inihahatid sa kanilang mga pahina ng paghahanap...sobra para sa isang na-crawl na web...

Ano ang layunin?

Mabilis na mga pahina na mukhang maganda? ADS na mabilis mag-load? Sinusubaybayan sa pinagmulan? Ang pattern para sa akin ay mukhang patayong pagsasama ng lahat ng mga bahagi na nagmumula sa user, sa pahina ng paghahanap, at sa nilalaman, papunta mismo sa search engine. Ang direksyon ay upang bumuo ng higit pa at higit pa sa paligid ng lubos na na-optimize na mga pader na hardin ( nanginginig).

Sana mali ako. Gusto kong makita ang amp na matunaw sa kung ano ang magiging HTML6 o isang katulad nito, tiyak na hindi ako gumagamit ng mga pang-eksperimentong bahagi. Karamihan sa website ay na-browse mula sa mobile. Ang paggawa ng pagkakaiba sa pagitan ng "desktop" at "mobile" sa termino ng bilis ay maaaring makapinsala kung ngayon ang mga tao ay nag-a-upgrade ng kanilang mga telepono nang mas mabilis kaysa sa kanilang mga desktop at laptop, upang ang kanilang telepono ay maaaring maging kanilang pinakamabilis na device. Ang mga pahina ay dapat na idinisenyo para sa kakayahang magamit sa lahat ng oras, ang "tumugon" na diskarte, na maaaring magpabigat sa pagbuo ng website nang husto, ay isang mas mahusay na trade-off pagkatapos ay magulong multi-bersyon na mga web page.

Meron ding Gemini

Kung gusto mo ng magaan na web page, ang Gemini protocol ay maaaring maghatid ng mga iyon, susuportahan ba ito ng mga pangunahing browser? Malamang hindi...dahil hindi ito makapaghatid ng mga ad..
