vindruvor för winetricks

En lagerbaserad installation för winetricks recept

Vin används för att köra Windows -program på Linux. Körtiden fungerar genom att återimplementera dlls att vara linuxkompatibel. Men programvara behöver inte bara basbiblioteken, det finns beroenden som måste installeras för varje programvara.

Wine stöder olika Windows -versioner ... (alla?), Därför är det nödvändigt att använda olika miljöer för att säkerställa kompatibilitet mellan programvara och Windows -versioner.

Vissa beroenden kan också vara oförenliga med varandra, varför användningen av vinprefix . Att använda vinprefix innebär att vanligtvis varje applikation har sin egen miljö, (som behållare), vilket betyder att alla beroenden måste installeras om för varje nytt program . Du har inte det också, olika program kan dela samma miljö, men som sagt kan problem som är svåra att felsöka uppstå.

Winetricks automatiserar installationen av beroenden (som .NET- och DirectX -paket), men även om det är skriptmässigt kan inställningar fortfarande ta ganska lång tid.

  • Förslag

Ostree används vid kassan versioner av operativa system . Och även om det huvudsakliga användningsfallet är att möjliggöra atomuppgraderingar för operativsystem installerade på barmetall, kan det användas för att bygga vinprefix med förinstallerade samlingar av beroenden, som flatpak använder samma metod för att dela körtider mellan olika applikationer, kan det faktiskt finnas olika flatpak vinstidför programvarufamiljer som delar huvuddelen av beroenden; den enda frågan som återstår är om det finns några licensproblem att distribuera sådan typ av körtider med vad som faktiskt skulle vara förinstallerad Windows -programvara . : S
