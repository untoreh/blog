我的节目设置

主要是我如何处理动漫和电视节目..

一开始我做了什么

我以前只是浏览 跟踪 ，并管理一个列表 观看 显示，和 掉线 我不喜欢的电视节目。然后我选择了每天观看的内容，然后流式传输或抢先下载（因为互联网连接很慢，我不得不在早上做，就像例行公事：P）。

一段时间后我切换到了什么

我用了 科迪 有一堆插件，但它总是一个错误，不一致，不断的插件升级，高内存使用，由发布迭代引起的无法解决的问题以及插件针对不同的 kodi 版本的事实...... Kodi 是一个巨大的野兽而且你必须学习如何处理它......我不会被打扰......它可能是电视的最佳选择，但如果你的环境是标准桌面，则低于标准。在 kodi 之后，我决定尝试 微服务 使例程自动化的方法。我开始使用相当多的服务......

  • [Sonar] 用于搜索版本

  • 弹性获取 用于更新曲目列表

  • qbt 或者 洪水 下载

  • 市场 字幕

  • 夹克 对于来源聚合

  • 有什么看的。

我从来没有选择一个看剧集的好应用，所以我只是用 mpv，首先使用 [https://github.com/StareInTheAir/mpv-trakt-sync-daemon] 切换到 trakt，然后切换到更稳定的 [https://github.com/iamkroot/trakt-scrobbler]。

该设置以单体方式在 docker 容器内运行。我没有采用 docker compose 来运行所有容器化的东西，比如 这个 ;优点是您可以单独更新应用程序......缺点是其他一切。无论如何，整个事情都应该在本地运行，因此命名空间分离的好处是毫无意义的，因为不存在扩展问题（最坏的情况需要 2GB 的 RAM）。事实上，我在我的日常 PC 上运行它而没有注意到任何减速。虽然后来我换了一个 骡子 服务器，这样我就可以在需要时有更多的自由和更少的麻烦来重新启动我的电脑。

这可以进一步简化吗？

当然。我更愿意在使用 gui 进行发布处理时失败，并且自己将事情粘在一起。我承认我仍然需要 下载器 （传输）和 球员（jellyfin），但我可以远离使用美杜莎，或者只是界面。我可以使用 Medusa python 模块作为框架来构建适合我需求的我自己的（超简化的）逻辑。 Flexget 也是这方面的一个选择。

我还管理 trakt 列表吗？

不，我切换到 simkl。它在各方面都优于 trakt。

  • 加载几乎所有东西都更快

  • 用户界面更干净，背后有很多想法。很难准确指出什么更好，但与 trakt 相比，我需要的功能总是很容易找到。

  • 与将“收藏”、“监视列表”和“监视”作为默认列表的 trakt 相比，分桶显示非常简单。使用 trakt 我不得不管理我不喜欢的节目的“删除”列表，而使用 simkl 我只是将它们设置为“不感兴趣”，这会自动将它们从“下一个播出”日历中删除。此外，浏览自动将节目视为“正在观看”，因此不需要“观看列表”。

  • 快速转发观看状态很容易，再次感谢 UI 和允许从其他服务或从 CSV 或 JSON 列表迁移的导入功能。

  • 发现新节目也更简单。在我不得不通过 trakt 日历查找新节目之前，我可以按日期查找 IMDB 数据库。 Simkl 自动区分回归节目的首映和新节目的首映。

仍然没有一个很好的自托管解决方案来跟踪电视节目， 弗洛克斯缺少我使用 simkl 的很多东西。显示发现不是我可以编写脚本的东西，因为我仍然需要一个可靠的界面来帮助我做出选择，所以看起来我暂时会使用 simkl。

动漫、电视剧还是电影？

大多数设置对电视节目和动漫很有用。我真的不看很多电影，除了一些受欢迎的电影，除了一些不错的 CGI 之外，我对它们没有太多期望。我发现电影的内容要么太快要么太空洞，有时他们试图告诉你一些东西，但一次又一次地完成，结果要么超过了顶部，要么错过了标记。

相反，电视节目节奏更慢，而且事情会更好地沉淀，但是如果我说我没有狂看某些节目，我会撒谎，而狂看电视节目感觉就像在看一场 非常 长电影。然而，即使您一次观看，在剧集之间有硬中断的扩展格式本身也能提供节奏。在制作节目时，频道事先知道他们有多少小时（通常至少 10 小时），因此他们有更多的回旋余地来根据故事改编剧本。对于电影，一切都在 1 小时 30 分到 2 小时之间乱七八糟地狭窄，很少到 3 小时。在时间预算有限的情况下正确剪辑场景，大多数情况下最终会导致一系列混乱的事件，很难跟踪，因为没有足够的 重量 在他们后面。

然而……如今的电视节目经常充斥着政治议程，这使得它们 不那么令人愉快.电视节目（和电影）的制作方式需要观众，目标人群。如果节目没有定义一个，那么它甚至不被考虑。

动漫来了。对我来说，动漫还没有失去艺术气息，要么是因为它们中的大多数是漫画改编，要么是因为没有关于目标人群的假设。也许与电视节目相比，制作动画的成本降低了 快速迭代 关于不同的想法。不管是什么原因，我发现自己越来越多地看动漫和动画节目而不是电视节目，因为后者一次又一次地辜负了我的期望。

PS：这里的措辞有点乱。你也可以有电视节目形式的动画节目和电影，但大多数时候我打算作为动画，动画系列的时长约为 20 百万，而电视节目的时长约为 40 米（至少是戏剧，而喜剧通常是20m 形式）。
