为什么 Windows 是无法修复的错误之山

Windows (11) 咆哮。

以下是窗户永远不会比流沙更好的原因列表。

它充满了遥测

操作系统 ping 多少个端点？多频繁？遥测可以完全禁用吗？可能不是。操作系统不是在禁用遥测的情况下开发的，它会导致一些奇怪的超时/停止行为，因为它期望 MS 端点是 可达.

原因不明的神秘错误代码

当出现问题时，所有 Windows 都会给你一个错误代码，几乎从来没有错误消息，如果它出现了，那么它是不可理解的，因为它引用了一些很可能是封闭源代码的运行时。

无处不在的 ACL

当然，ACL 是更好的安全模型。当您收到权限被拒绝错误时，祝您好运找出需要哪些权限和权限。

驱动程序

Windows 无法识别此设备...那么您如何解决此问题？请问我可以戳一个漂亮的地方吗？我什至不提 司机签名.

用户状态

用户数据存储在%APPDATA%local 和roaming .如果我浏览这两个文件夹我看不到明确的定义，一些应用程序将缓存存储在一个文件夹中，尽管它是 配置 其他人则相反……Windows甚至有适当的约定吗？

Windows 没有 PATH

每个二进制文件通常都存储在自己的目录中（连同所有供应商的依赖项）。如果您尝试概括路径，您最终会得到一个存储二进制文件的数千个目录的列表，至少有scoop.

Windows 更新被推送...非常愤怒

显然可能会发生 Windows 推送错误的更新，而不是拉回，或者甚至不针对它们被推送到的平台的更新！也没有提到 Windows 升级是原子的，所以每次你执行升级时，你都是在掷骰子。

用户界面

windows shell 更像是一个kiosk 与 UI 相比，您可以按下按钮并单击内容，只要得到 Windows UI 团队的批准，就没有适当的快捷方式 API。每个配置键盘快捷键的应用程序都是一个 hack。
