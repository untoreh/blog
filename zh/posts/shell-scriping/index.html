外壳划线

关于shell脚本的思考

重击/ zsh shell 脚本可以完成工作。它也很丑陋。也许是有意为之。你想看的越少，脚本就会越短。我不喜欢使用 bash/zsh 作为交互式 shell，弥补良好体验所需的插件数量令人生畏，每个额外的插件都是可能导致 shell 减速的陷阱。尽管如此，psprint 完成的工作量 齐尼特和朋友们 非常令人印象深刻（如果您想将 zsh 用作交互式 shell，这也是插件管理器的唯一明智选择）。然而，它并没有解决所有这些加载的插件都是 非共享 （可能）许多 shell 进程之间的内存。

鱼壳 是我在撰写本文时首选的外壳。已安装插件的数量非常少，因为它可以完成您希望从 shell 中获得的大部分功能 盒子外面 .脚本编写比 zsh 慢一些，但没有人关心脚本编写速度，所以很好。带有结束语句而不是花括号的 ML 传统（如 Julia）的语法很好。编写脚本比 bash/zsh 更冗长，但它也更具可读性。诸如过程替换、字符串插值和管道之类的事情最终会变得更加冗长。它有 全球的 变量，在所有正在运行的进程之间共享，以及 普遍的 跨会话共享的变量，这使事情变得更容易，而无需写下所有内容和源健忘的脚本。

精灵语是一个支持很多东西的外壳，比如一个守护进程或网络模式，它允许它远程发送命令......？我认为它太多了，我开始称它为臃肿。恕我直言，shell 应该力求简单，并且是自包含的。我认为 shell 应该被设计为处理这样​​的场景，即它们是进程产生的显着次数，从头到尾的行为一致，并且是自包含的，以便它们可以优雅地处理崩溃。因此，尽管有所有其他功能（如结构化数据）...守护进程不适合我。

离子壳 是 几乎 posix 来自 redox 的 shell，它也适用于 unix，目标是 还 在 Windows 上工作，虽然它不起作用 ATTOW。它的目标只是成为一个更好的外壳，所以它从 posix 中汲取了好处并添加到它。好像是专注于提供一个好的脚本语言，但是交互体验不是很好，fish shell 把我宠坏了，我不回壳了 头等舱 互动性。

油壳 目标是取代 bash，这是值得称道的，但它只针对没有 Windows 的 unix 系统。到目前为止，它一直专注于修复 bash 兼容层，未来计划专注于交互部分。考虑到 shell 的交互部分尚未编写，这让我对它的未来有点怀疑，虽然我希望它不会，但我认为它可能会在实现预期目标之前失去动力。此外，缺乏 Windows 支持也不理想。

皮疹 支持结构化数据的外壳 球拍物体，当我更多地查找球拍时，我会尝试一下，尽管我过去使用 [ 外壳 ] 不是很好，因为 外壳 仅适用于 emacs，不能很好地作为 系统外壳 ...除非你的系统是 emacs（尽管 emacs 是我的主要编辑器，我确实写 一些 lisp，它不是我的操作系统）。 （在这里我还应该提到 Julia REPL，它是任何编程语言的最佳 repl）。

电源外壳 Powershell，主要的 Windows shell 也适用于 linux 和 macs，这要归功于跨平台 网 .但它太冗长，太慢，而且可能（未经验证）到处都是 窗户主义 ;这对我来说是一个很大的不。

怒壳 ...当然我们把最好的留到最后。这可能是我的下一个 shell；对交互性的一流关注，仔细考虑每个 shell 内置和跨平台，以及结构化文本。

不在此列表 ATTOW 中的 Shell 可能被认为不够有趣。
