为什么文本的 LLM 如此之大？

跑步 gpt-6b 需要 12GB 内存 gpt-20b 更。

相比 稳定扩散 需要 3GB 到 5GB。

什么会影响运行模型的内存要求？参数的数量，因为每个参数都是一个浮点数（16/32/64 位）。

一张图片抵得上千字？在压缩知识方面，图像是否比语言更好？也许。也可能图像通常比文本更模糊。

什么是好的输出？

假设输出集是固定的（整个宇宙）。我们可以用不同的短语来描述图像，并且所有这些短语都是可接受的输入，或者我们可以接受来自相同文本输入的多个图像作为有效输入。

而要从某些文本输入中获得可接受的文本输出，它必须遵循更严格的规则输入的含义。这意味着更少的输入映射到完整的输出集。

换句话说，文本到图像的 LLM 具有更高的 主观性 比文本到文本的 LLM，因为什么被分类为可接受的输出取决于我们作为图像“肉袋”分类器的局限性。我们从文本中考虑一个好的输出图像，如果该图像承载了我们提示的总体含义，我们就会留下很多“解释”。而当我们阅读文本时，我们希望它遵循严格的语法规则并具有意义。

那么为什么文本转图像变小了呢？

我们可以掩盖图像中的缺陷，因为不是每个人都能画图和创作图像，但每个人都能阅读和写出体面的散文。 LLM 的效用反映在我们完成 LLM 旨在完成的任务的能力上。我们做某事的能力越差，LLM 的复杂性就越低，它需要的参数就越少。换句话说，我们是垃圾画家！我们对“好”文本到图像模型的标准设置得较低。如果每个人都是非常好的图像分析器/制作者，那么我们对什么是好图像的要求更严格，那么文本到图像模型必须比文本到文本模型大得多，因为需要多少分析才能真正剖析图像的所有含义。

题外话理解文本或图像意味着什么

当然，文本到图像和文本到文本的 LLM 都没有真正的意义，但是用图像伪造意义比用文本更容易，因为图像是比文本更高层次的抽象。文本是一种串行通信形式，其中传输新信息连接一系列具有某种意义的标记（称为段落）。

图像反而具有分形属性，其意义不依赖于序列先前某个位置的某个已知簇，而是依赖于 同时 画布中多个簇的位置。

一种处理图像的算法

要“理解”图像，您必须在不知道特定顺序的情况下创建集群，仅遵循颜色差异，并对这些集群进行分类（在实体、形状、深度、闪电等方面），在每个合成级别（从 1像素到整个画布表面），然后为所有生成的聚类级别之间的每个可能关系分配权重。这些至少需要 3 个大型模型，如果你想专门处理分类过程，还需要另外 1 个。

机器学习自由度和硬件

令人遗憾的是，当不超过 5 倍的改进允许在每个机器上本地运行非常有效的 LLM 时，摩尔定律（或光刻，如果谈到 ram 数量）就停滞不前了！
