操作系统跳跃

我使用过的操作系统的简要总结

当我得到我的第一台个人电脑时，它配备了 Windows XP（13 岁）。经过几次混乱后不得不学习如何重新安装 Windows...它不像今天那样即插即用...我对 linux 的第一次体验是使用 PCLinuxOS，这显然是 仍在继续 因为它提供了完整的图形安装。之后我直接进入 拱形Linux .我玩了一段时间的 Arch Linux（比如让 gnome UI 看起来像 MacOS 或 Windows Vista 侏儒外观模组 )，然后又回到 Windows 因为游戏.. 从 Windows 我管理了一段时间运行 centos/debian 的个人 linux 服务器。几年后回到Linux，首先我尝试 高山 ，但它的存储库（还？）不能满足桌面体验（pulseaudio 仍然在 测试 回购）。

然后我试过了 无效的Linux 有一段时间，但我正在使用 穆尔 没有很好的支持包明智的基于版本，所以我在容器中运行 void linux plus 应用程序 i3wm 基于环境。但是容器化的图形应用程序还没有普及，像 flatpak snapd 或 firejail 这样的解决方案还没有普及甚至稳定，所以我切换到 Ubuntu，并设置了一个 ZFS RAID-Z（ 危险地生活 ) 根安装。作为桌面环境我还是保留了 i3wm，然后去 摇摆 导致wayland，然后是KDE等离子，因为我不能再为必须是的systemd单元文件的垃圾而烦恼了 用户 设法通过任何平铺 WM 获得最小的工作预期设置。

KDE 很棒，我认为它是桌面环境的最前沿，但是它也很大，并且有很多应用程序缺乏良好的维护，所以我的建议是不要太努力地坚持只使用 K* 的 GUI 环境和只需使用有效的方法，KDE 对 GTK 样式有很好的支持。 KWin 具有非常好的自定义选项，您可以绑定热键使其表现得非常接近 浮动优先 平铺 WM。如果你想要一个 平铺优先 平铺 WM 然后有几个扩展，但是在 Wayland 上效果不佳（在撰写本文时）。

最近我又切换回 Windows (10).. 再次 , 不主要 因为游戏 因为我还得找时间和动力再玩一次，但有点出于好奇 WSL 基于 linux 环境加上图形应用程序的良好集成（ 工作组 ）。在撰写本文时，遗憾的是 Windows 10 体验实际上是从 linux KDE 降级。窗户与其 以鼠标为中心 UX 似乎没有任何面向深度键盘快捷键定制的内容。不清楚是否通过 视窗文化 快捷方式被认为是一个安全漏洞，或者它只是 windows shell 的一个严重限制，它强制通过注册表覆盖进行自定义。 电动玩具 要满足一个希望让环境适应他们习惯的人的所有需求，还有很长的路要走（不确定我是否可以被视为高级用户......）。尽管如此，我可以非常灵活，并且通过一些第三方程序，环境已经变得可以接受了。
