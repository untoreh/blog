uvas para vinhos

Uma configuração baseada em camadas para receitas de winetricks

Vinho é usado para executar programas do Windows no Linux. O tempo de execução funciona reimplementando dlls para ser compatível com Linux. No entanto, o software não precisa apenas das bibliotecas básicas, existem dependências que precisam ser instaladas para cada parte do software.

Wine suporta diferentes versões do Windows ... (todas?), Portanto é necessário usar diferentes ambientes para garantir a compatibilidade entre o software e as versões do Windows.

Algumas dependências também podem ser incompatíveis entre si, portanto, o uso de prefixos de vinho . Usar prefixos de vinho significa que normalmente cada aplicativo tem seu próprio ambiente, (como contêineres), o que significa que todas as dependências tem que ser reinstalado para cada novo programa . Você também não tem, programas diferentes podem compartilhar o mesmo ambiente, mas, como declarado, podem surgir problemas difíceis de depurar.

O Winetricks automatiza a instalação de dependências (como pacotes .NET e DirectX), mas, mesmo se por script, as configurações ainda podem levar algum tempo.
