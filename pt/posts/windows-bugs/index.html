Por que o Windows é uma montanha incorrigível de bugs

Desabafo do Windows (11).

Aqui está uma lista de razões pelas quais as janelas nunca serão melhores do que areias movediças.

Está cheio de telemetria

Quantos endpoints o SO faz ping? com que frequência? A telemetria pode ser totalmente desativada? Provavelmente não. O sistema operacional não é desenvolvido com a telemetria desabilitada, ele incorrerá em alguns comportamentos estranhos de tempo limite / paralisação porque espera que os pontos de extremidade do MS sejam alcançável.

Códigos de erro crípticos com causas não especificadas

Tudo o que o Windows fornece quando algo dá errado é um código de erro, quase nunca uma mensagem de erro, e se isso acontecer é incompreensível porque faz referência a algum tempo de execução que provavelmente é de código fechado.

ACL em todos os lugares

Claro, ACL é o melhor modelo de segurança. Boa sorte para descobrir onde e quais permissões são necessárias quando você recebe um erro de permissão negada.

Motoristas

O Windows não reconheceu este dispositivo... Então, como você corrige isso? Posso cutucar em algum lugar por favor? nem estou mencionando assinaturas de motorista.

Estado do usuário

Os dados do usuário são armazenados em%APPDATA%local eroaming . Se eu navegar pelas duas pastas, não vejo definições claras, alguns aplicativos armazenam o cache em uma pasta, apesar de ser para configuração outros fazem o oposto... Será que o Windows ainda tem convenções apropriadas?

O Windows não tem PATH

Cada binário geralmente é armazenado em seu próprio diretório (junto com todas as dependências fornecidas). Se você tentar generalizar o caminho, acabará com uma lista de milhares de diretórios onde os binários são armazenados, pelo menos háscoop.

As atualizações do Windows são enviadas... com muita raiva

Aparentemente, pode acontecer que o Windows envie atualizações ruins, que recuam ou atualizações que nem são para a plataforma para a qual são enviadas! Também não há menção de atualizações do Windows serem atômicas, então toda vez que você faz uma atualização, você está jogando os dados.

IU

O shell do Windows é mais como umkiosk do que uma interface do usuário, você pode pressionar botões e clicar em coisas, desde que seja sancionado pela equipe de interface do usuário do Windows, não há API adequada para atalhos. Todo aplicativo para configurar atalhos de teclado é um hack.
