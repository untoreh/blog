Torne o Windows suportável

Uma lista de ferramentas para tornar o Windows mais parecido com o Linux

Desde que eu mudou para o Windows 10 com WSL Decidi fazer uma lista de softwares que precisava usar para aproximar o Windows do meu fluxo de trabalho normal do Linux. Vamos deixar claro que a IU do Windows é uma droga, está presa no90s e tudo deve ser clicado e arrastado com o ponteiro do mouse. Não sei se é um cultura coisa (os desenvolvedores do Windows devem gostar muito do mouse), ou para alvo demográfico ("clique naquela coisa!" é fácil de explicar), ou por causa de melhores práticas de segurança (se dermos muito controle ao usuário, o malware irá explorá-lo) ou apenas o Shell do Windows sendo feito de uma pilha de dívidas tecnológicas que torna mais difícil adicionar funcionalidade para usuários avançados. O fato é que coisas simples no Linux precisam de soluções complicadas no Windows.

Desktops virtuais

No Linux, todo gerenciador de janelas tem alguma forma de área de trabalho virtual, as janelas foram construídas recentemente com o W10. A questão é que, no momento da escrita, você só pode alternar para a esquerda e para a direita com as teclas de atalho, e não há teclas de atalho para ir para a área de trabalho X. biblioteca vd mas eu não encontrei software que faça uso dele, ao invés apenas AHK[1] scripts com implementações pobres como "loop até que estejamos no VD correto".

A falta de atalhos adequados para desktops virtuais integrados ao Windows me fez forçar o uso de software de terceiros, dos quais há muitos, eu escolhi dexpot como faz todas as coisas que eu preciso e muito mais (eu ficaria feliz com apenasgo to X emove window to X atalhos). O único problema era que dexpot não é capaz de vincular chaves já vinculadas ... de alguma forma, AHK é capaz de substituir as chaves usadas por outros aplicativos, enquanto dexpot não, presumo que seja porque ele depende de uma API diferente do Windows (ou [dll]). A solução alternativa é remapear os atalhos que eu queria (por exemplo,Win+1 ) com um atalho gratuito que pode ser definido no dexpot (por exemploWin+Shift+F1)

Mapeamento de teclado

Powertoys permite mapear teclas e atalhos, de modo que eu possa mapear CapsLock-> LeftControl e RightControl-> LeftControl. E atalhos comoWin+hjkl para flechas. Muito ruimWin+l é um atalho padrão do Windows para bloquear a tela e não pode ser remapeado ... uma chave de registro substitui os atalhos e corrige isso ... até que você atualize o Windows ... então você deve aplicar a correção a cada inicialização!

Lançador

O inicializador de aplicativos fornecido pelo powertoys é bastante cheio de recursos, não usa muito mais memória do que o KRunner e é rápido e responsivo. A única reclamação é que de vez em quando ele perde o foco, mas é mais provável que seja um problema com o controle maluco do Windows (como no pervertido) de qual janela deve estar focada em um determinado momento.

Revestimento

Eu mudei do balanço para o KDE no Linux, então eu não era muito um amante de janelas lado a lado. Powertoys tem FancyZones, que é uma ligeira melhoria em relação aos Snaps básicos do Windows que a janela oferece, pois oferece lacunas e layouts. No entanto, ainda falta a coisa mais importante que torna o tiling útil, que é as regras para se aplicar a janelas correspondentes, mas isso também precisaria de suporte para desktops virtuais ... e já mencionamos seu estado atual. Na comparação, o KWin permite combinar janelas com definições muito complexas. O FancyZones contorna o problema de regras preservando as posições das janelas nas sessões, mas não investiguei como isso funciona bem com diferentes desktops virtuais.

Arrastando janelas

As janelas só podem ser arrastadas da barra de título e um Utilitário precisa ser usado para movê-los com uma combinação de atalhos.

Monitor de sistema

Com o KDE, você tem widgets de plasma para exibir informações do sistema, embora os widgets de plasma tendam a consumir muita memória, então eu não os usei muito. Com o Windows não há um utilitário embutido para isso, depois de tentar alguns utilitários, resolvi usar Monitor de tráfego que dá rede, cpu, memória e, recentemente, foram adicionados GPU e temps. O resultado é um belo retângulo de baixo perfil exibido na barra de tarefas:

O ponto de bônus para o Windows é que meu UPS é reconhecido e é mostrado na área da bandeja, então não precisa de configuração adicional, enquanto no linux PORCA existem problemas de motoristas.

Serviços do Sistema

Muitos não gostaram do systemd quando ele veio para o kernel do linux .. Com o Windows eu comecei a sentir falta dele .. Serviços do Windows não é realmente algo orientado para os usuários como o systemd. O Windows tem o Agendador de tarefas para lidar com coisas que parecemone-shotarquivos de unidade, mas novamente sua interface (ou a falta dela) é terrível. Para nossa sorte, não precisamos de daemons personalizados no Windows, já que a maioria dos aplicativos do Windows que usamos são apenas correr na inicialização tipo de lógica e não precisa de configurações mais avançadas. Na verdade, além do lixo de ferramentas ( suspirar ) para fazer o Windows funcionar como um ambiente moderno, apenas outros aplicativos nativos do Windows em uso são o navegador ( Raposa de fogo ) já que a aceleração da GPU do navegador é ruim no WSL (bem, mesmo no Linux nativo ...) e no player de vídeo ( mpv ) .. e jogos é claro ...

Pacotes

Eu não gosto achocolatado como requer privilégios de administrador, sempre procuro primeiro os pacotes scoop, pois eles estão instalados na pasta do usuário, o que é mais conveniente e consistente e simplifica os backups.

WSL / g

Fui induzido a mudar para o Windows do recente WSLg atualizar. Quais drivers mesa necessários compilados com od3d12 suporte de back-end paraopengl . O Windows fornece uma camada do Ubuntu de visualização da comunidade, mas optei por Arch Linux uma vez que já havia um pacote AUR para mesa com d3d12 . Como o WSL atualmente não oferece suporte ao systemd, acabei usando Supervisor para gerenciar alguns serviços. Para ter certeza de que o supervisor está ativo, há uma verificação no perfil do shell para um arquivo de bloqueio que deve ser criado emtmpfs se o supervisor foi iniciado anteriormente.

WSL suporta apenasext4 sistemas de arquivos, para usar outros sistemas de arquivos, você tem que montar a partição (ou o disco) diretamente dentro da VM WSL. No entanto, eles devem ser montados manualmente como administradordas janelas. Para automatizar isso, podemos usar o agendador de tarefas que permite ignorar o prompt do UAC executando tarefas com o maior privilégio . Podemos executar um wsl--mount para montar nossos discos / partições desejados e então executar mount o sistema de arquivos de dentro do Linux. Porque o meu/home/ reside em um montadobtrfs sistema de arquivos eu tenho que montá-lo automaticamente, então usamos/etc/fstab para mapear nossa partição (porLABEL ) para/home . Porque usamos nix comnix-env precisamos vincular a montagem de nossa loja nix a/nix em cada inicialização wsl, e porque/tmp não está ligadotmpfs temos que criar uma montagem de sobreposição que montatmpfs sobre o existente/tmp , para preservar arquivos importantes, especificamente os arquivos X11, necessários para se comunicar com o servidor X.

Essas montagens são executadas com um script, que deve ser executado depois de nós montamos o disco dentro do linux. Precisamos de uma tarefa que seja executada após a tarefa de montagem ser concluída e, como o WSL usa VMs diferentes para diferentes usuários do Windows, o script deve ser executado NÃO com os privilégios mais altos (caso contrário, ele seria montado na VM do Administrador).

Para garantir que o WSLg funcione, temos que nos certificar de que o XDG TEMPO DE EXECUÇÃO DIR é definido, uma vez que é diferente e localizado (por padrão) em/mnt/wlsg/runtime-dir . É assim que as janelas no wayland se parecem, você pode ver que uma vez que as decorações das janelas do wayland podem ser desenhadas tanto pelo compositor quanto pelo aplicativo, eles mantêm a configuração do seu tema GTK / QT, então você acaba com um tema nativo do linux olhando a janela dentro de um shell do Windows ... o que é um pouco desorientador no início.

Emulador de terminal

No linux eu estava usando gatinha , já que era um dos emuladores de terminal mais novos com aceleração de GPU[2]e um modo daemon um tanto estável que permite várias janelas com a mesma instância. No entanto, no Windows, experimentei alguns terminais baseados em Windows, como o Windows Terminal, fluent terminal, wsltty, conemu, mas eventualmente mudei para wezterm . Apesar de ser um terminal mais recente, ele tem todos os recursos que você gostaria de um emulador de terminal:

  • guias e painéis, de modo que você não precise do tmux

  • uma multiplexação (modo remoto sem cabeça), de modo que você não precise do tmux

  • modo de cópia de texto com atalhos de teclado vi, de modo que você não precise do tmux

  • seleção rápida, um modo de dica para selecionar rapidamente objetos de texto na tela de rolagem para trás

  • comandos padrão para novas guias

  • Muito boa personalização de mapas de teclas, além do fato de que você não pode usar Shift / Ctrl / Alt / Esquerda / Direita como modificadores diferentes.

Com o Windows, também modifiquei meu fluxo de trabalho um pouco, de modo que não preciso de um terminal com modo daemon que ofereça suporte a várias janelas separadas, usando um terminal suspenso. Para fazer wezterm suspenso depois de tentar muitos scripts AHK suspensos, acabei descobrindo 1 que funcionou muito bem. Uma alternativa melhor, entretanto, seria Windows-terminal-terremoto, mas eu tive problemas com ele se comportando mal com os desktops virtuais gerenciados por dexpot, enquanto mitty-quake-console parece ser compatível com desktops virtuais dexpot. Provavelmente, quando a experiência dos desktops virtuais do Windows integrados melhorar, mudarei para o Windows-terminal-quake. O WezTerm também é multiplataforma, o que significa que posso continuar usando-o com uma instalação Linux nativa. Foi uma descoberta realmente feliz para este switch Windows + WSL, uma vez que o Kitty ainda não oferece suporte ao Windows, e as outras alternativas de terminal, tiveram alguns aborrecimentos bastante importantes que Eu não conseguia chegar a um acordo com.
