scripting shell

Gânduri asupra scripturilor shell

Bash/ zsh scriptul shell face treaba. De asemenea, este destul de urât. Poate că este intenționat. Cu cât vrei să o privești mai puțin, cu atât scripturile vor fi mai scurte. Nu sunt un fan al utilizării bash / zsh ca un shell interactiv, cantitatea de pluginuri necesare pentru a compensa o experiență plăcută este descurajantă și fiecare plugin suplimentar este o capcană pentru posibilele încetiniri ale shell-ului. Cu toate acestea, cantitatea de muncă realizată de psprint cu zinit & prieteni este foarte impresionant (și, de asemenea, este singura alegere sensibilă pentru un manager de pluginuri dacă doriți să utilizați zsh ca un shell interactiv). Cu toate acestea, nu rezolvă faptul că toate pluginurile încărcate sunt nepartajat memorie printre (probabil) multe procese shell.

Coajă de peșteeste cochilia mea preferată în momentul scrierii. Cantitatea de pluginuri instalate este foarte mică, deoarece face cele mai multe lucruri pe care le-ați dori de la un shell afara din cutie . Scriptarea este oarecum mai lentă decât zsh, dar nimănui nu îi pasă de viteza de scriptare, așa că este bine. Sintaxa este frumoasă a patrimoniului ML (cum ar fi Julia) cu declarații de final, în loc de aparate dentare. Scrierea scripturilor este mai detaliată decât bash / zsh, totuși este, de asemenea, mult mai lizibilă. Lucruri precum înlocuirea procesului, interpolare de șiruri și conducte sfârșesc puțin mai detaliat. Are global variabile, partajate între toate procesele care rulează și universal variabile, partajate între sesiuni, ceea ce face lucrurile mai ușoare fără a fi nevoie să scrieți totul și să obțineți scripturi uitate.

Elf este un shell cu suport pentru multe lucruri, cum ar fi un demon sau un mod web care îi permite să trimită comenzi de la distanță ...? Cred că este prea mult și încep să-l numesc umflat. IMHO un shell ar trebui să se străduiască pentru simplitate și să fie autonom. Cred că shell-urile ar trebui să fie concepute pentru a face față scenariilor în care sunt procese care au generat o cantitate vizibilă de ori, care se comportă constant de la început până la sfârșit și care sunt autonome, astfel încât să poată face față accidentelor cu grație. Deci, în ciuda tuturor celorlalte caracteristici (cum ar fi datele structurate) ... demonii nu sunt de folos pentru mine.

Ion Shell este aproape posix shell din redox, care funcționează și pe unix, cu scopul de de asemenealucrează pe Windows, deși nu funcționează ATTOW. Scopul său este să fie pur și simplu un shell mai bun, așa că ia binele de la posix și îl adaugă. Se pare că se concentrează pe oferirea unui limbaj de scriptare bun, dar nu o experiență interactivă foarte bună, coaja de pește m-a stricat și nu mă întorc la o coajă fără clasa întâi interactivitate.

Coajă de ulei are scopul de a înlocui bash, ceea ce este lăudabil, cu toate acestea vizează doar sistemele unix fără Windows la vedere. S-a concentrat pe remedierea stratului de compatibilitate bash până acum, cu planuri viitoare de concentrare pe partea de interactivitate. Având în vedere faptul că partea interactivă a shell-ului nu a fost încă scrisă, mă face puțin îndoielnică cu privire la viitorul său și, deși sper că nu, îmi place că ar putea pierde abur înainte de a-și atinge obiectivele propuse. De asemenea, lipsa suportului pentru Windows nu este ideală.

Eczemă un shell cu suport pentru date structurate care funcționează cu Rachetă obiecte, când voi căuta mai multe rachete, o voi încerca, deși experiența mea trecută cu [ emacs shell ] nu este chiar frumos, deoarece eshell funcționează numai în emacs și nu funcționează frumos ca un shell de sistem ... cu excepția cazului în care sistemul dvs. este emacs (în ciuda emacs este editorul meu principal și eu scriu niste lisp, nu este sistemul meu operativ). (Aici ar trebui să menționez și Julia REPL, cea mai bună replică pentru orice limbaj de programare).

Powershell Powershell, shell-ul Windows principal funcționează și pe Linux și Mac, datorită platformei încrucișate NET . Dar este prea amănunțit, lent și probabil (nu verificat) plin de ferestrele ; Este un mare nu pentru mine.

Nu shell... bineînțeles că ne economisim tot ce este mai bun pentru ultimul. Acesta este probabil următorul meu acces la shell; atenție de primă clasă pentru interactivitate, examinare atentă a fiecărei platforme încorporate și cross-platform, și text structurat.

Obuzele care nu se află în această listă ATTOW au fost probabil considerate că nu sunt suficient de interesante.
